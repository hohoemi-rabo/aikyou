# 04 — システムプロンプト構築

## 目的

`state` を人が読める形に整形し、システム指示＋連結ナレッジ＋state を**固定順序**で組み立て、プロンプトキャッシュが効く形にする。

## 背景

連結ナレッジは約4万トークン規模。毎ターン再送するとコストが嵩むため**プロンプトキャッシュ必須**。キャッシュ対象（不変部）を**先頭に固定**することがヒットの条件。

## プロンプトの順序（厳守 / REQUIREMENTS.md §8・§7.1）

1. **システム指示 ＋ 連結ナレッジ**（毎回不変）← ここを `cache_control: { type: 'ephemeral' }` でキャッシュ
2. 現在の `state`（毎セッション更新）
3. そのセッションの会話履歴（毎ターン増える）

1 の順序・内容を絶対に動かさない。

## システム指示の骨子（REQUIREMENTS.md §7.1）

- 「{title}（{game_version}）を一緒に遊ぶ少し詳しい先輩ゲーマー（相棒）」という役割。
- 絶対ルール：必ず `{game_version}` の仕様で答える。他の版（SFC/GBC/スマホ/HD-2D）を混ぜない。
- 答えを一方的に全部言わず、会話相手として的確に答える。

## 対象ファイル

- `src/lib/prompt.ts`（`formatState()` / `buildSystemBlocks()` 等）
