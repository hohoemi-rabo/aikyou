- 複数ゲーム対応／ナレッジ差し替えUI

## フェーズ1からの方針変更点

- **会話履歴**：フェーズ1は「継続性は state のみ・履歴はメモリのみ」だったが、ふりかえり閲覧と冒険ログ出力のため `messages` に永続化する。ただし**継続性の担保は引き続き state**であり、ライブのチャットはセッション中メモリ保持のまま（履歴の自動プリロードはしない）。
- 履歴保存は**ベストエフォート**：保存に失敗してもチャット応答は妨げない（`/api/chat` のストリーム完了後に保存し、失敗は `console.error`）。
